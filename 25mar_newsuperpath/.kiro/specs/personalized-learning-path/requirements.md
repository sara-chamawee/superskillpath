# เอกสารข้อกำหนด (Requirements Document)

## บทนำ

Prototype สำหรับระบบเส้นทางการเรียนรู้แบบ Personalized ที่ช่วยให้ผู้ใช้สามารถจัดการทักษะและเลือกทักษะที่ต้องการพัฒนา โดยได้รับการนำทางจาก AI ผ่านระบบแชทแบบสนทนา (Chat-based) เพื่อพัฒนาทักษะอย่างเป็นขั้นตอน ผู้ใช้ทุกคนมีบทบาทเดียวกัน สามารถทั้งจัดการข้อมูลทักษะและเรียนรู้ทักษะได้ ระบบใช้ข้อมูลทักษะตั้งต้นจากไฟล์ #[[file:aidlc-skill-feat-skill-based-execution 3/Skills Name.md]] และใช้ข้อมูล Course content จากไฟล์ #[[file:seed-data/skill-content-mapping.md]] เป็นข้อมูลคอร์สเรียนที่ผูกกับแต่ละทักษะ เพื่อให้ AI แนะนำคอร์สที่เกี่ยวข้องระหว่างการเรียนรู้

## อภิธานศัพท์ (Glossary)

- **System**: ระบบเส้นทางการเรียนรู้แบบ Personalized ทั้งหมด
- **Skill_Manager**: โมดูลที่รับผิดชอบการสร้าง แก้ไข และลบข้อมูลทักษะ
- **Skill**: ทักษะที่ประกอบด้วย ชื่อ รายละเอียด เกณฑ์การวัดผล และ checklist
- **Assessment_Criteria**: เกณฑ์การวัดผลของทักษะ ใช้ประเมินว่าผู้เรียนบรรลุทักษะแล้วหรือไม่
- **Checklist**: รายการตรวจสอบที่ผู้เรียนต้องทำให้ครบเพื่อถือว่าผ่านทักษะนั้น
- **Skill_Catalog**: รายการทักษะทั้งหมดที่เปิดให้ผู้ใช้เลือก
- **Seed_Data**: ข้อมูลทักษะตั้งต้นที่นำเข้าจากไฟล์ Skills Name.md ประกอบด้วย ชื่อทักษะ คำนิยาม เกณฑ์การวัดผล และ checklist
- **Content_Mapping**: ข้อมูล mapping ระหว่าง Skill กับ Course content ที่นำเข้าจากไฟล์ Skill x Content ประกอบด้วย Course ID, ชื่อคอร์ส, ผู้สอน, ระยะเวลา, ประเภทการประเมิน (Submit Assignment File / Chat to Assess)
- **Course**: คอร์สเรียนที่ผูกกับทักษะ ประกอบด้วย ชื่อคอร์ส ผู้สอน ระยะเวลา และผู้ให้บริการเนื้อหา
- **AI_Chat_Engine**: โมดูล AI ที่ทำหน้าที่สนทนากับผู้ใช้ผ่านระบบแชท เพื่อนำทางการเรียนรู้ ให้เนื้อหา ถามคำถาม มอบแบบฝึกหัด และติดตามความก้าวหน้า
- **Chat_Session**: เซสชันการสนทนาระหว่างผู้ใช้กับ AI_Chat_Engine สำหรับการเรียนรู้ทักษะ
- **Learning_Path**: เส้นทางการเรียนรู้ที่ AI สร้างขึ้น ประกอบด้วยขั้นตอน เนื้อหา และกิจกรรมที่เรียงลำดับ
- **Progress_Tracker**: โมดูลที่ติดตามความก้าวหน้าของผู้ใช้ตาม checklist ของทักษะ
- **User**: ผู้ใช้งานระบบที่สามารถทั้งจัดการข้อมูลทักษะและเรียนรู้ทักษะตามเส้นทางที่ AI สร้างให้

## ข้อกำหนด (Requirements)

### ข้อกำหนดที่ 1: การนำเข้าข้อมูลทักษะตั้งต้นจากไฟล์ Seed Data

**User Story:** ในฐานะ User ฉันต้องการให้ระบบนำเข้าข้อมูลทักษะจากไฟล์ Skills Name.md เป็นข้อมูลตั้งต้น เพื่อให้มีทักษะพร้อมใช้งานตั้งแต่เริ่มต้นโดยไม่ต้องสร้างทักษะใหม่ทั้งหมด

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. WHEN ระบบเริ่มทำงานครั้งแรกและยังไม่มีข้อมูลทักษะใน Skill_Catalog, THE System SHALL นำเข้าข้อมูลทักษะจากไฟล์ #[[file:aidlc-skill-feat-skill-based-execution 3/Skills Name.md]] เป็น Seed_Data
2. THE System SHALL แปลงข้อมูลแต่ละแถวในตาราง Seed_Data ให้เป็น Skill โดยจับคู่คอลัมน์ Skills Name เป็นชื่อทักษะ, Skill Definition เป็นรายละเอียดทักษะ, Areas of Measurement เป็น Assessment_Criteria, และ Checklist เป็นรายการ Checklist
3. WHEN การนำเข้า Seed_Data เสร็จสมบูรณ์, THE System SHALL แสดงทักษะทั้งหมดที่นำเข้าใน Skill_Catalog
4. IF ไฟล์ Seed_Data มีข้อมูลที่ไม่สมบูรณ์ในแถวใดแถวหนึ่ง, THEN THE System SHALL ข้ามแถวนั้นและบันทึก log แจ้งเตือนว่าแถวใดมีข้อมูลไม่สมบูรณ์
5. WHEN ระบบมีข้อมูลทักษะอยู่แล้วใน Skill_Catalog, THE System SHALL ไม่นำเข้า Seed_Data ซ้ำ

### ข้อกำหนดที่ 1.5: การนำเข้าข้อมูล Course Content ที่ผูกกับทักษะ

**User Story:** ในฐานะ User ฉันต้องการให้ระบบนำเข้าข้อมูลคอร์สเรียนที่ผูกกับแต่ละทักษะ เพื่อให้ AI สามารถแนะนำคอร์สที่เกี่ยวข้องระหว่างการเรียนรู้ได้

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. THE System SHALL นำเข้าข้อมูล Content_Mapping จากไฟล์ Skill x Content โดยจับคู่คอร์สกับทักษะที่ตรงกัน
2. THE System SHALL บันทึกข้อมูล Course แต่ละรายการ ประกอบด้วย: Course ID, ชื่อคอร์ส, ผู้ให้บริการเนื้อหา, ชื่อผู้สอน, ระยะเวลา, และ Domain ของทักษะ
3. THE System SHALL บันทึกประเภทการประเมินของแต่ละทักษะ (To Do List Type) ว่าเป็น "Submit Assignment File" หรือ "Chat to Assess"
4. WHEN ทักษะมีไฟล์ To-Do List หรือ Simulation แนบมา, THE System SHALL บันทึกลิงก์ไฟล์นั้นไว้กับทักษะ
5. WHEN AI_Chat_Engine สร้าง Learning_Path, THE AI_Chat_Engine SHALL ใช้ข้อมูล Course ที่ผูกกับทักษะนั้นเป็นเนื้อหาอ้างอิงในการแนะนำคอร์สเรียนให้ผู้ใช้
6. WHEN User อยู่ใน Chat_Session, THE AI_Chat_Engine SHALL สามารถแนะนำคอร์สเรียนที่เกี่ยวข้องกับขั้นตอนการเรียนรู้ปัจจุบัน พร้อมระบุชื่อคอร์ส ผู้สอน และระยะเวลา

### ข้อกำหนดที่ 2: การจัดการทักษะ

**User Story:** ในฐานะ User ฉันต้องการสร้างและจัดการข้อมูลทักษะ เพื่อให้มีทักษะที่ถูกต้องและครบถ้วนให้เลือกเรียน

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. WHEN User กดปุ่มสร้างทักษะใหม่, THE Skill_Manager SHALL แสดงฟอร์มที่ประกอบด้วยฟิลด์: ชื่อทักษะ, รายละเอียดทักษะ, เกณฑ์การวัดผล (Assessment_Criteria), และ Checklist
2. WHEN User กรอกข้อมูลทักษะครบทุกฟิลด์ที่จำเป็นและกดบันทึก, THE Skill_Manager SHALL บันทึกทักษะใหม่และแสดงทักษะนั้นใน Skill_Catalog
3. WHEN User เลือกแก้ไขทักษะที่มีอยู่, THE Skill_Manager SHALL แสดงฟอร์มแก้ไขที่มีข้อมูลเดิมของทักษะนั้นพร้อมให้แก้ไข
4. WHEN User กดบันทึกหลังแก้ไขทักษะ, THE Skill_Manager SHALL อัปเดตข้อมูลทักษะและแสดงข้อมูลที่อัปเดตแล้วใน Skill_Catalog
5. WHEN User กดลบทักษะ, THE Skill_Manager SHALL แสดงข้อความยืนยันก่อนดำเนินการลบ
6. WHEN User ยืนยันการลบทักษะ, THE Skill_Manager SHALL ลบทักษะออกจาก Skill_Catalog
7. IF User กดบันทึกโดยไม่กรอกฟิลด์ที่จำเป็น, THEN THE Skill_Manager SHALL แสดงข้อความแจ้งเตือนระบุฟิลด์ที่ยังไม่ได้กรอก

### ข้อกำหนดที่ 3: การกำหนด Checklist ของทักษะ

**User Story:** ในฐานะ User ฉันต้องการกำหนด checklist สำหรับแต่ละทักษะ เพื่อให้มีเกณฑ์ที่ชัดเจนในการวัดว่าผู้เรียนบรรลุทักษะแล้ว

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. WHEN User สร้างหรือแก้ไขทักษะ, THE Skill_Manager SHALL อนุญาตให้ User เพิ่มรายการ Checklist ได้ตั้งแต่ 1 รายการขึ้นไป
2. THE Skill_Manager SHALL บันทึกรายการ Checklist แต่ละรายการโดยประกอบด้วยข้อความอธิบายสิ่งที่ผู้เรียนต้องทำ
3. WHEN User ลบรายการ Checklist ออกจากทักษะ, THE Skill_Manager SHALL อัปเดตรายการ Checklist ของทักษะนั้นทันที
4. IF User พยายามบันทึกทักษะโดยไม่มีรายการ Checklist, THEN THE Skill_Manager SHALL แสดงข้อความแจ้งเตือนว่าต้องมี Checklist อย่างน้อย 1 รายการ

### ข้อกำหนดที่ 4: การแสดงรายการทักษะและการเลือกทักษะ

**User Story:** ในฐานะ User ฉันต้องการเห็นรายการทักษะทั้งหมดและเลือกทักษะที่ต้องการพัฒนา เพื่อให้ AI สร้างเส้นทางการเรียนรู้ที่เหมาะกับฉัน

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. THE Skill_Catalog SHALL แสดงรายการทักษะทั้งหมด (รวมทักษะจาก Seed_Data และทักษะที่สร้างเพิ่มเติม) โดยแสดงชื่อและรายละเอียดของแต่ละทักษะ
2. WHEN User เลือกทักษะจาก Skill_Catalog, THE Skill_Catalog SHALL แสดงรายละเอียดทั้งหมดของทักษะนั้น รวมถึง Assessment_Criteria และ Checklist
3. WHEN User กดเลือกทักษะที่ต้องการพัฒนา, THE System SHALL บันทึกทักษะที่ User เลือกไว้ในโปรไฟล์ของ User
4. THE System SHALL อนุญาตให้ User เลือกทักษะได้มากกว่า 1 ทักษะ
5. WHEN User เลือกทักษะเสร็จแล้วและกดเริ่มเรียน, THE System SHALL เปิด Chat_Session กับ AI_Chat_Engine สำหรับทักษะที่เลือก

### ข้อกำหนดที่ 5: การสร้างเส้นทางการเรียนรู้ผ่านแชท AI

**User Story:** ในฐานะ User ฉันต้องการให้ AI สร้างเส้นทางการเรียนรู้และนำทางฉันผ่านการสนทนาแบบแชท เพื่อให้การเรียนรู้เป็นธรรมชาติและ AI ขับเคลื่อนการเรียนรู้ให้มากที่สุด

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. WHEN User เริ่ม Chat_Session สำหรับทักษะที่เลือก, THE AI_Chat_Engine SHALL เริ่มต้นการสนทนาโดยแนะนำตัว อธิบายภาพรวมของทักษะ และสอบถามพื้นฐานความรู้เดิมของ User เกี่ยวกับทักษะนั้น
2. THE AI_Chat_Engine SHALL สร้าง Learning_Path ที่ประกอบด้วยขั้นตอนการเรียนรู้ที่เรียงลำดับจากง่ายไปยาก โดยอ้างอิงจาก Assessment_Criteria และ Checklist ของทักษะนั้น
3. THE AI_Chat_Engine SHALL นำเสนอเนื้อหาและกิจกรรมในแต่ละขั้นตอนผ่านข้อความแชทแทนการแสดงเป็นหน้า UI แบบดั้งเดิม
4. WHEN User มีทักษะที่เลือกมากกว่า 1 ทักษะ, THE AI_Chat_Engine SHALL สร้าง Chat_Session แยกสำหรับแต่ละทักษะ
5. THE AI_Chat_Engine SHALL ขับเคลื่อนการสนทนาเชิงรุก โดยถามคำถาม ให้แบบฝึกหัด และแนะนำขั้นตอนถัดไปโดยอัตโนมัติ โดย User ไม่จำเป็นต้องนำทางด้วยตนเอง

### ข้อกำหนดที่ 6: การนำทางผู้เรียนผ่านแชท AI

**User Story:** ในฐานะ User ฉันต้องการให้ AI ช่วยนำทางและให้คำแนะนำระหว่างเรียนผ่านการสนทนา เพื่อให้ฉันเข้าใจเนื้อหาและทำกิจกรรมได้สำเร็จ

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. WHEN User อยู่ใน Chat_Session, THE AI_Chat_Engine SHALL ให้เนื้อหาการเรียนรู้และคำแนะนำผ่านข้อความแชทตามขั้นตอนใน Learning_Path
2. WHEN User ถามคำถามเกี่ยวกับเนื้อหาในขั้นตอนปัจจุบัน, THE AI_Chat_Engine SHALL ตอบคำถามโดยอ้างอิงจากบริบทของทักษะและขั้นตอนที่กำลังเรียน
3. WHEN User ทำกิจกรรมในขั้นตอนปัจจุบันเสร็จ, THE AI_Chat_Engine SHALL แจ้งผลและแนะนำให้ User ไปยังขั้นตอนถัดไปผ่านข้อความแชท
4. IF User ไม่เข้าใจเนื้อหาในขั้นตอนปัจจุบัน, THEN THE AI_Chat_Engine SHALL อธิบายเนื้อหาเพิ่มเติมในรูปแบบที่แตกต่างออกไป เช่น ยกตัวอย่าง ใช้การเปรียบเทียบ หรือแบ่งเนื้อหาเป็นส่วนย่อย
5. WHEN User ไม่ตอบกลับในระหว่าง Chat_Session เป็นเวลานาน, THE AI_Chat_Engine SHALL ส่งข้อความกระตุ้นเพื่อเชิญชวนให้ User กลับมาเรียนรู้ต่อ

### ข้อกำหนดที่ 7: การติดตามความก้าวหน้าของผู้เรียน

**User Story:** ในฐานะ User ฉันต้องการเห็นความก้าวหน้าของตัวเอง เพื่อให้รู้ว่าฉันเรียนรู้ไปถึงไหนแล้วและเหลืออะไรอีกบ้าง

#### เกณฑ์การยอมรับ (Acceptance Criteria)

1. THE Progress_Tracker SHALL แสดงสถานะของแต่ละรายการใน Checklist ว่าทำสำเร็จแล้วหรือยังไม่สำเร็จ
2. WHEN User ทำรายการ Checklist สำเร็จผ่านการสนทนากับ AI_Chat_Engine, THE Progress_Tracker SHALL อัปเดตสถานะของรายการนั้นเป็น "สำเร็จ" และคำนวณเปอร์เซ็นต์ความก้าวหน้าใหม่
3. WHEN User ทำ Checklist ครบทุกรายการของทักษะ, THE Progress_Tracker SHALL แสดงสถานะว่า User บรรลุทักษะนั้นแล้ว
4. THE Progress_Tracker SHALL แสดงเปอร์เซ็นต์ความก้าวหน้าโดยรวมของแต่ละทักษะที่ User เลือก
5. WHEN User ถาม AI_Chat_Engine เกี่ยวกับความก้าวหน้า, THE AI_Chat_Engine SHALL สรุปสถานะ Checklist และเปอร์เซ็นต์ความก้าวหน้าผ่านข้อความแชท
